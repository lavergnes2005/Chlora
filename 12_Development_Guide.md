# CHLORA PROJECT

# 12 --- Development Guide

**Version:** 1.1 **Authority:** Support **Status:** Living

## Purpose

Explains how the project is maintained and how contributions become
canon.

## Documentation Hierarchy

1.  Project Constitution
2.  Canon Bible
3.  Reference Documents
4.  Support Documents
5.  Conversation history

## Core Rule

**One concept. One authoritative location.**

Move information rather than duplicate it.

## Canon Workflow

1.  Proposal
2.  Discussion
3.  Approval
4.  Update Canon Bible
5.  Update affected reference documents
6.  Record rationale in the Design Journal

## Contribution Guidelines

-   Follow the Constitution.
-   Respect document ownership.
-   Prefer real science before speculation.
-   Preserve established terminology.
-   Mark proposals clearly until approved.

## Repository

-   Markdown is the source of truth.
-   Git is the project history.
-   One file per document.
-   Freeze documents between formal documentation releases.
